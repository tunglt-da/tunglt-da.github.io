ETL Pipeline with Airflow
Problem Statement

Manual and fragmented ETL processes made it difficult to maintain consistent, up-to-date datasets for real-time operational reporting across multiple warehouses.

Solution

I designed and implemented a robust ETL pipeline using Apache Airflow that:

  • Orchestrates end-to-end workflows to build and update datasets supporting operational dashboards and hourly reporting systems.
  • Extracts data from multiple sources (APIs, Redash, BigQuery, flat files).
  • Performs complex transformations, anomaly detection, and data quality checks.
  • Loads curated datasets into analytical layers for Power BI and Looker Studio.
  • Implements real-time error handling with alerts sent via Google Chat and Slack.
  • Runs on a lightweight environment via WSL (Windows Subsystem for Linux), with a manually configured Airflow setup for better control, performance tuning, and flexibility without using containerization.
Technologies Used
Apache Airflow Python BigQuery PostgreSQL Slack API Google Chat API WSL
Results & Impact
  • Increased pipeline reliability from 65% to 99.5% uptime.
  • Enabled consistent hourly data refreshes for real-time dashboards.
  • Reduced data processing time by 30%, improving report timeliness.
  • Eliminated silent failures with automated monitoring and alerts.
  • Enhanced overall data quality and operational decision-making.
